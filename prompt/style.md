# Coding Style Guide

This project adheres to the following coding style guidelines:

1.  **C++ Google Style Guide:** All C++ code must follow the official [Google C++ Style Guide](https://google.github.io/styleguide/cppguide.html). This includes naming conventions, formatting, class design, and other best practices.

2.  **English Comments:** All comments within the code, including header comments, inline comments, and Doxygen-style documentation, must be written in English.
